# Product Guidelines

## Prose Style
- **Clarity and Precision:** Use clear, unambiguous language. Avoid jargon where possible, but use industry-standard terms when necessary for fraud investigation.
- **Action-Oriented:** Write button labels and instructions starting with strong verbs (e.g., "Assign Case", "Flag Transaction").
- **Professional Tone:** Maintain an objective and professional tone suitable for a risk management context.

## Branding
- **Color Palette:** Use neutral tones for the base UI with semantic colors for indicators (e.g., Red for high-risk, Yellow for medium-risk, Green for safe).
- **Typography:** Ensure high readability for dense data sets. Prefer sans-serif, monospaced fonts for numerical data and IDs.

## UX Principles
- **Information Density:** Optimize for high information density without visual clutter. Analysts need to see relevant data simultaneously.
- **Efficiency:** Minimize clicks for common actions (e.g., bulk assigning cases, quick-filtering queues).
- **Contextual Awareness:** Always provide clear context for alerts or flags, allowing users to drill down into the transaction history or linked entities directly from the summary view.
- **Accessibility:** Ensure high contrast and keyboard navigability for power users.
