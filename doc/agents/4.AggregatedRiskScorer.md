# 🎛️ Agent #4 — Aggregated Risk Scorer
_Sentinel (Agent ThinkING)_

The **Aggregated Risk Scorer** ingests the outputs from upstream agents and produces a **single, calibrated risk score (0–100)**, a **severity label** (LOW/MED/HIGH/CRITICAL), and a **recommended action** (ALLOW/REVIEW/CHALLENGE/BLOCK).  
It **does not execute actions** — it only recommends. Execution is handled by **Agent #5 – Action Executor**.

Upstream inputs:
- **Agent #1 — Transaction Pattern Analyzer** → `risk_score`, `flags`, `reasoning`
- **Agent #2 — Behavioral Risk Agent** → `behavioral_risk_score`, `flags`, `feature_contributions`
- **Agent #3 — Evidence Builder** → `evidence_summary`, `combined_flags`
- **(Optional) AML/Compliance Agent** → `aml_score`, `flags`

---

## ✨ Outputs (What this agent returns)

- `final_risk_score` (0–100) — calibrated composite
- `severity` — `LOW | MED | HIGH | CRITICAL`
- `recommended_action` — `ALLOW | REVIEW | CHALLENGE | BLOCK` (suggestion only)
- `source_contributions` — per‑source contribution to final score
- `explanation` — short, deterministic reason string
- `policy_version`, `model_version`, `version` — for auditability

---

## 🧱 Step‑by‑Step Plan

### 1) Define the contract (ADK‑friendly JSON)

**Request (input)**
```json
{
  "transaction_id": "tx_123",
  "customer_id": "cust_456",

  "pattern_agent": {
    "risk_score": 72,
    "flags": ["AMOUNT_SPIKE","GEO_MISMATCH"],
    "version": "pattern-v1.0.0"
  },

  "behavioral_agent": {
    "behavioral_risk_score": 64,
    "flags": ["NEW_DEVICE","UNUSUAL_TIME"],
    "feature_contributions": { "amount_zscore": 3.2, "geo_distance_km": 520.5 },
    "version": "behavior-v1.0.0"
  },

  "evidence_agent": {
    "evidence_summary": "High amount vs pattern, new device off-hours, abnormal distance.",
    "combined_flags": ["AMOUNT_SPIKE","NEW_DEVICE","UNUSUAL_TIME","GEO_MISMATCH"],
    "version": "evidence-v1.0.0"
  },

  "aml_agent": {
    "aml_score": 55,
    "flags": ["PEP_MATCH_PENDING"],
    "version": "aml-v1.0.0"
  }
}
````

**Response (output)**

```json
{
  "final_risk_score": 83,
  "severity": "CRITICAL",
  "recommended_action": "BLOCK",
  "source_contributions": {
    "pattern_agent": 40,
    "behavioral_agent": 32,
    "aml_agent": 11
  },
  "explanation": "Weighted blend: amount spike + new device + geo mismatch; AML moderate.",
  "policy_version": "policy-2026-02-24",
  "model_version": "agg-v1.0.0",
  "version": "aggregator-v1.0.0"
}
```

***

### 2) Normalize & blend scores (transparent logic)

*   **Normalization**: convert incoming scores to **\[0..1]** internally.
*   **Missing data**: if an upstream agent is absent, **redistribute weights** proportionally or use a configurable default.
*   **Weighted blend (baseline)**:
    *   Default weights (configurable):  
        `pattern = 0.40`, `behavioral = 0.40`, `aml = 0.20`
    *   `blend = Σ (w_i * s_i)` (all `s_i` in \[0..1])
*   **Risk boosters (optional)**: small non‑linear bump when high‑risk flag combos co‑occur (e.g., `AMOUNT_SPIKE + NEW_DEVICE + GEO_MISMATCH`), capped to prevent runaway scores.
*   **Calibration**: piecewise linear or Platt scaling to map blended score → **\[0..100]**.
*   **Deterministic explanation**: one‑line string built from top contributing sources and high‑priority flags.

> Guardrail: **monotonicity** — increasing any upstream score (others fixed) must **not decrease** the final risk score.

***

### 3) Severity bands & recommendation (policy‑driven)

*   Severity thresholds (configurable):
    *   `0–34 → LOW`
    *   `35–59 → MED`
    *   `60–79 → HIGH`
    *   `80–100 → CRITICAL`
*   Recommendation mapping (configurable):
    *   `LOW → ALLOW`
    *   `MED → REVIEW`
    *   `HIGH → CHALLENGE`
    *   `CRITICAL → BLOCK`

These live in `config/policy.yaml` so risk ops can tune without redeploys.

***

### 4) HTTP API

    POST /score/aggregate     # body = Request JSON → returns Response JSON
    GET  /healthz

*   Strict JSON schema validation (400 on invalid).
*   Include `policy_version`, `model_version`, and `version` in every response.

***

### 5) Configuration files

*   `config/policy.yaml`
    *   `weights`: per‑source weights (pattern/behavioral/aml)
    *   `thresholds`: severity bands
    *   `boosters`: risky flag combos and caps
    *   `calibration`: parameters for mapping to \[0..100]
    *   `recommendations`: severity → action map

*   `config/flags.yaml` *(optional)*
    *   Defines priority ordering of flags for explanations.

Version both files and surface their versions in logs and responses.

***

### 6) Observability & audit

*   **Structured logs**: `txnId` (hashed), incoming scores, normalized values, final score, severity, recommendation, `policy_version`, `model_version`, `duration_ms`
*   **Metrics**: score histogram, severity counters, missing‑agent rate, p95 latency, error rate
*   **Tracing**: correlation ID from the Orchestrator

***

### 7) Testing strategy

*   **Unit tests**: normalization, blending, boosters, calibration
*   **Property tests**: monotonicity, bounded outputs \[0..100]
*   **Golden tests**: canonical LOW/MED/HIGH/CRITICAL scenarios with frozen configs
*   **Missing‑agent tests**: absent AML or Pattern/Behavioral inputs
*   **Contract tests**: request/response schema validation

***

### 8) Performance targets

*   p95 **< 30–60 ms** (math + formatting only)
*   Stateless; horizontally scalable

***

### 9) Future enhancements

*   Per‑segment policies (e.g., retail/SME/VIP)
*   Automated calibration via recent outcomes
*   Explainability with contribution breakdown by **flag** (not just by source)
*   Sandboxed policy experiments (A/B policy IDs)

## 🔒 Security notes

*   Reject unknown fields and invalid schemas
*   Avoid raw PII in logs; hash IDs for correlation
*   Include `policy_version`, `model_version`, `version` in responses & logs
*   Use IAM & least‑privilege permissions in deployment

## ☁️ Deploy (Cloud Run example)

```bash
# Build container with Jib
mvn -pl agents/aggregated-scorer-java -DskipTests package \
  jib:build -Djib.to.image=eu.gcr.io/$GCP_PROJECT_ID/aggregated-scorer:$(git rev-parse --short HEAD)

# Deploy
gcloud run deploy aggregated-scorer \
  --image eu.gcr.io/$GCP_PROJECT_ID/aggregated-scorer:$(git rev-parse --short HEAD) \
  --region $GCP_REGION \
  --allow-unauthenticated=false \
  --set-env-vars "APP_ENV=prod,POLICY_FILE=/app/config/policy.yaml"
```
